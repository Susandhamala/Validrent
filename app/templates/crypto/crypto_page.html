RSA-2048 Key Generation
Asymmetric · On every registration

When a user registers, generate_rsa_keypair() is called to produce a 2048-bit RSA key pair. The same call is used internally to create the Demo CA's key at app startup. Public exponent 65537 is the standard choice — it is prime and makes encryption fast.

RSA-PSS + SHA-256 Signature
Asymmetric · Agreement signing step

Both landlord and tenant digitally sign the SHA-256 hash of the agreement document. PSS (Probabilistic Signature Scheme) uses a random salt so the same document signed twice gives different bytes — preventing replay attacks. The signature is stored in the DB and re-verified on every QR scan.

Sign — used in code
app/services/crypto_service.py line 96–104 
def rsa_sign(private_key_pem, data_hex):
    private_key = load_private_key(private_key_pem)
    data_bytes = bytes.fromhex(data_hex)
    signature = private_key.sign(
        data_bytes,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    return base64.b64encode(signature).decode('utf-8')
Verify — used in code

def rsa_verify(public_key_pem, data_hex, signature_b64):
    try:
        public_key = load_public_key(public_key_pem)
        public_key.verify(
            signature,
            data_bytes,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            ),
            hashes.SHA256()
        )
        return True
    except Exception:
        return False

AES-256-GCM Encryption
Symmetric · Files · Photos · Chat

AES-256-GCM provides confidentiality (nobody can read the ciphertext) and authenticity (built-in 128-bit authentication tag detects tampering). A fresh random 12-byte nonce is generated for every encryption call — reusing a nonce would be catastrophic for GCM security.

Agreement file encryption

    nonce   = os.urandom(12)          # fresh per file
    aesgcm  = AESGCM(aes_key)
    ciphertext = aesgcm.encrypt(nonce, plaintext, None)
    # writes ciphertext to disk

SHA-256 Hash
Hash function · Integrity everywhere

SHA-256 turns any data into a fixed 64-character hex string. Change one byte of the input and the hash changes completely. Used to fingerprint documents, photos, chat messages, and the PDF — and to derive the server master key.

Document hash (tamper detection)
app/services/crypto_service.py line 52–53 
def sha256_hash_bytes(data: bytes) -> str:
    return hashlib.sha256(data).hexdigest()
Chunked file hash (large files)
app/services/crypto_service.py line 44–49 
def sha256_hash_file(file_path):
    h = hashlib.sha256()
    with open(file_path, 'rb') as f:
        for chunk in iter(lambda: f.read(8192), b''):
            h.update(chunk)
    return h.hexdigest()

RSA-OAEP + SHA-256
Asymmetric · Key wrapping (hybrid encryption)

OAEP (Optimal Asymmetric Encryption Padding) encrypts small payloads (like a 32-byte AES key) with a recipient's RSA public key. This is the "hybrid encryption" pattern: AES-256-GCM encrypts the large data, RSA-OAEP encrypts the AES key itself.

Encrypt a key with RSA public key

def encrypt_rsa_oaep(public_key_pem, plaintext_bytes):
    public_key = load_public_key(public_key_pem)
    ciphertext = public_key.encrypt(
        plaintext_bytes,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    return base64.b64encode(ciphertext).decode()

    plaintext = private_key.decrypt(

X.509 Certificates (PKI)
Standard · Identity binding per user

The Demo CA issues a signed X.509 certificate to every user on registration, binding their name, email, and RSA public key. The CA itself is an ephemeral RSA-2048 self-signed certificate created at app startup. Certificates can be revoked — revocation is checked on every QR scan.

    cert = (

        .subject_name(subject)                # name + email
        .issuer_name(ca_cert.subject)          # signed by Demo CA
        .public_key(public_key)               # user's RSA-2048 pubkey
        .not_valid_after(now + timedelta(days=365))  # 1 year
        .sign(ca_key, hashes.SHA256(), default_backend())
    )
Certificate verification (QR scan)
app/services/certificate_service.py line 88–110 
def verify_certificate(cert_pem):
    cert = x509.load_pem_x509_certificate(cert_pem.encode())
    # 1. Check expiry
    if cert.not_valid_after_utc < now:
        return False, 'Certificate has expired'
    # 2. Verify CA signature on the cert
    ca_cert.public_key().verify(
        cert.signature,
        cert.tbs_certificate_bytes, ...
    )
    return True, 'Valid'
